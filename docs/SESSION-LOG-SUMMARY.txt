KEY DECISIONS MADE
================================================================================

1. ESLint v9 flat config
   Rationale: Modern, explicit globals management for Nitro auto-imports

2. Prettier integration
   Rationale: Enforces consistent formatting (120 char, single quotes, trailing commas)

3. Husky + lint-staged
   Rationale: Catch issues before commit; uses postinstall to avoid Nitro conflict

4. Bun + bun:test
   Rationale: Native TypeScript support, fast execution

5. GitHub Actions CI/CD
   Rationale: Lint → Format → Test → Coverage on every PR

6. Custom skills (3)
   Rationale: Reusable workflows specific to project needs

7. OpenSpec workflow
